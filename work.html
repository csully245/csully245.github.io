Tesla

Systems Validation Intern, Vehicle Software

May to July 2022

During my summer at Tesla, I was part of the LV Body Controls Validation group.
My project was to create a tool to automate speaker tests while maintaining audio fidelity.
Past solutions for audio tests used a standard microphone, which was subject to background noise, making it nearly unusable.
In my solution, I developed a device using discrete electronics to record the signal straight from the amp and compare it to the expected signal.
The project included digital signal processing in Python and large-scale software development work
to integrate the tool into the wider Tesla LV Body Controls testing suite.
The tool has been adopted for emergency call and pedestrian warning system testing across all vehicle platforms.


Visteon

Test Equipment Development Intern

May to August 2021

Visteon is an automotive electronics supplier specializing in dashboard electronics.
Originally strictly under the Ford umbrella, it now serves OEMs globally, though my work was all within Ford.
I worked on two projects at Visteon: validating the Ford Escape instrument cluster and developing a voltage standards testing tool.
The instrument cluster validation project involved operating environmental test equiment and
working with software and hardware engineering teams to address specification failures.
I also developed a test environment for a set of specifications using Vector CANalyzer and CAPL.
The voltage standards device was designed to apply the maximum specified voltage to each pair of pins on a given device.
Previously, this was done manually with a power supply and probes. My project automated this, saving hours of technician time per test.
The project involved Arduino embedded code and CVI test/GUI control code.
